Symmetric elimination
Suppose we have a symmetric (or hermitian) matrix  A
 . Let's proceed with Gaussian elimination a little differently.

A = magic(4);
A = A + A' + eye(4)
A =

    33     7    12    17
     7    23    17    22
    12    17    13    27
    17    22    27     3
We start as before, with a unit lower triangular  L1
  that puts zeros in the first column.

L1 = eye(4);  L1(2:4,1) = -A(2:4,1)/A(1,1);
A1 = L1*A
A1 =

   33.0000    7.0000   12.0000   17.0000
         0   21.5152   14.4545   18.3939
         0   14.4545    8.6364   20.8182
         0   18.3939   20.8182   -5.7576
That didn't change the first row of  A
  at all. If we were to transpose  A1

This result is symmetric, which is no accident. In fact we can get here simply by

A1 = L1*A*L1'
A1 =

   33.0000         0         0         0

         0   18.3939   20.8182   -5.7576
As long as we apply elimination symmetrically on the left and right, we preserve the symmetry of  A
  as the elimination proceeds.

L2 = eye(4);  L2(3:4,2) = -A1(3:4,2)/A1(2,2);

A2 =

   33.0000         0         0         0
         0   21.5152         0         0
         0         0   -1.0746    8.4606
         0         0    8.4606  -21.4831

   33.0000         0         0         0
         0   21.5152         0         0
         0         0   -1.0746   -0.0000
         0         0         0   45.1258
This matrix is (up to roundoff) symmetric and upper triangular, hence diagonal. In summary, we have a symmetric factorization of  A
 :

L = inv(L1)*inv(L2)*inv(L3);  D = diag(diag(A3));

   2.8422e-14

HPD matrices and the Cholesky factorization
If in addition to symmetry/hermitianness we also know that the quadratic form  x‚àóAx
  is positive for all nonzero  x
 , then  A
  is said to be positive definite (often written SPD or HPD). By taking  x=ek
 , we can see that  akk>0

The resulting Cholesky factorization takes the form  A=R‚àóR
  for an upper triangular  R
  with positive diagonal entries. Of course we don't need to compute both an upper and a lower triangle, and as a result it takes asympototically half as many flops as standard  LU
  factorization. Furthermore, the factorization is backward stable without pivoting.
